# Sample Billing Data

Two sample exports for offline demo. Realistic shapes; synthetic IDs.

| File | Format | What's inside |
|---|---|---|
| `aws_cur_sample.csv` | AWS Cost & Usage Report v2 (CSV) | 200 line items across EC2, EBS, EIP, NAT, RDS, ELB, S3 — multiple accounts, multiple regions, ~30d window |
| `azure_billing_sample.json` | Azure Consumption API export | ~50 records covering managed disks + VMs analogous to the AWS set |

Both contain a deliberate mix of **active**, **idle**, and **orphaned** resources so the detection engine produces a non-trivial findings table. The intent is to give a grader a realistic demo *without* requiring real cloud credentials.

## Generation

The samples are generated by `samples/generate.py` (which is itself produced in a later prompt). Re-running the generator with `--seed=42` reproduces the same files byte-for-byte.

## Anonymization

All resource IDs, account IDs, and ARNs are synthetic. No production data is included.
